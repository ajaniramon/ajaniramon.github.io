Testing

ése gran desconocido...

Tres tipos

  • Test unitarios
  • Test de integración
  • Test de aceptación

Test unitarios

  • El componente bajo test no habla con otros componentes ni sistemas externos
  • En caso de hacerlo, hay que mockear el componente
  • Los sistemas externos no se mockean, sino que se devuelve el resultado que estos generarían
  • Han de ser instantáneos
  • Los test no deben depender de otros test

Test de integración

  • Son aquellos test donde el componente bajo test habla con componentes o sistemas externos
  • Los sistemas externos pueden no ser los reales, pero siempre hay que tener un sistema externo lo más parecido a producción
  • Han de ser idempotentes
  • Los test no deben depender de otros test

Test de aceptación

  • Se pactan antes de empezar el desarrollo con el cliente
  • Realmente son test de integración de un proceso completo
  • Se plasman en FitNesse
  • El usuario debe entender la wiki de FitNesse y ser capaz de modificar los valores para realizar diferentes pruebas de aceptación

Consideraciones con Goldcar

Microservicio base

  • Creado por Alten
  • Sirve para añadir funcionalidades que ya existen en Spring... y soporte al Scheduler
  • La librería se descarga del repo de Goldcar y se incluye manualmente
  • Ejemplo en directo
  • Tutorial del setup

Scheduler

  • Es una aplicación independiente creada por Alten.
  • Permite automatizar llamadas a nuestra API y recoger el resultado de estas
  • Se integra con el microservicio base
  • Hay que realizar una configuración previa en la BD de Sigger
  • Las instrucciones para esto están en el manual de operaciones de la SMO
